# /plausibleba

Produce a complete business architecture bundle from any input — transcript, charter,
existing spreadsheet, or verbal description. Runs three phases in sequence and delivers
a single downloadable `ba-skills-bundle.json` at the end.

## Usage

```
/plausibleba
/plausibleba [brief description of the business or initiative]
```

Attach any supporting files before or after invoking the command.

## What this command does

Runs the PlausibleBA stack as a single guided session:

```
Phase 1 — Capability Map    What the organisation can do
Phase 2 — Concept Model     What the organisation manages
Phase 3 — Value Stream      How the organisation delivers value
                                    ↓
                         ba-skills-bundle.json
```

Each phase has **one checkpoint** where you review and confirm before proceeding.
Each phase renders an interactive visualisation inline as its primary output.
No downloads until the bundle is complete.

## What you get at the end

- **Interactive visualisations** for all three artefacts (rendered inline during the session)
- **`[organisation]-ba-skills-bundle.json`** — complete bundle for VCC import
- **XLSX workbooks** for any artefact (on request)

## Session flow

### Opening
Claude will ask for a brief description of your business or initiative (if not provided),
then state the plan:

> *"We'll build three artefacts in sequence — a Capability Map, a Concept Model, and a
> Value Stream. Each takes 5–10 minutes. At the end I'll package everything into a bundle
> you can load directly into VCC. Let's start with capabilities."*

### Phase 1 — Capability Map (~5–10 min)
- Establishes context and core business objects
- Drafts L1/L2 hierarchy → **Checkpoint 1A** (confirm before proceeding to L3)
- Derives L3 capabilities → **Checkpoint 1B** (confirm before rendering)
- Renders interactive treemap
- Carries business objects forward into Phase 2

### Phase 2 — Concept Model (~5 min)
- Seeds from business objects already identified in Phase 1
- Classifies each as Party / Resource / Record
- Maps key relationships → **Checkpoint 2** (confirm before rendering)
- Renders interactive concept graph
- Carries typed objects forward into Phase 3

### Phase 3 — Value Stream (~5–10 min)
- Identifies trigger, recipient, and terminal outcome
- Drafts 4–8 stages with entry/exit states
- Maps Phase 1 capabilities to stages → **Checkpoint 3** (confirm before rendering)
- Renders interactive stage view
- Flags any capabilities unused across all stages

### Bundle
After Checkpoint 3 is confirmed:
- Packages all three artefacts into `[organisation]-ba-skills-bundle.json`
- Offers XLSX downloads for any or all artefacts
- Provides VCC import instructions

## Individual skills

The three skills are also available as standalone commands for targeted work:

- `/capability-map` — Phase 1 only
- `/concept-model` — Phase 2 only (works best after `/capability-map`)
- `/value-stream` — Phase 3 only (works best after both)
